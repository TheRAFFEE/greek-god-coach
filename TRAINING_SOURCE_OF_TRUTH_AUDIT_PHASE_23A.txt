EXECUTIVE SUMMARY

The application currently has multiple competing training sources of truth:

1. src/lib/training-planner.ts
   - New canonical Phase 22 source.
   - Not wired yet.

2. src/app/page.tsx
   - Independently computes today’s workout, today’s run, planned distance, scheduled run, daily prescription, completion state, and Train display state.

3. src/lib/home-command-center.ts
   - Independently derives today’s run, run distance/duration, Home training model, Home completion status, and calls legacy evaluateTraining().

4. src/lib/training-engine.ts
   - Legacy session composer.
   - Creates warmup/workout/run/cooldown blocks, status, duration, warnings, priorities.
   - Overlaps heavily with DailyTrainingSession.

5. src/lib/coach-engine.ts
   - Legacy recommendation generator.
   - Creates daily workout prescription, running recommendation, readiness-adjusted workout, next-set and post-workout recommendations.

6. src/lib/seed-data.ts
   - Current plan data and fallback workout selector.
   - Contains hardcoded 12-week schedule, run/mobility prescriptions embedded as workout exercises, and fallback to workouts[0].

7. src/app/page.tsx Train UI + run logger
   - Uses TrainingEngineResult, scheduledRun, and plannedRunDistance.
   - Contains defaulting logic that can turn run duration/distance assumptions into logging defaults.

Observed bug paths
- Repeated “Hold: 3 mi”: page.tsx passes staleRunLabel: "Hold: 3 mi" into getTodayRunForDate(), then nextRunLabel and getScheduledRunForTraining() can carry that label forward.
- Home and Train disagreeing: Home receives homeCommandCenter.trainingEngineResult, while Train also has an internal fallback evaluateTraining() call with different fake/default progression and goal inputs.
- Run duration becoming reps: seed workouts encode "35-45 min" as Exercise.prescribedReps; legacy engines parse/filter this text differently.
- Mobility interpreted as workouts: seed data encodes mobility as exercises; legacy lift detection often means anything not run/walk/sprint/interval.
- Duplicate logging/completion behavior: Train creates workout sessions directly in page.tsx; run logging is separate via TrainRunLogger -> saveTrainRunLog(); completion is independently derived by Home, Train, Daily Check-In, and Weekly Review.
