# Training Classification Audit V1

**Phase:** 23C
**Status:** Validation-only audit
**Scope:** Seed training plan classification against Training Rules Constitution V1
**Runtime impact:** None — no Home, Train, Log, planner, adapter, engine, UI, or seed-plan behavior changed by this audit

---

## Executive Summary

- Seed workouts audited: **84**
- Lift Day: **48**
- Run Day: **12**
- Long Run Day: **12**
- Recovery Day: **12**
- Current planner disagreements found: **12**
- Intentional seed-plan Hybrid Days under Constitution V1: **0**

Main finding: all 12 Wednesday Zone 2 + Mobility + Core workouts should classify as **Run Day**, but current planner output creates both `lift` and `run` blocks because core work is treated as a workout/lift prescription.

---

## Audit Method

This audit applies `docs/TRAINING_RULES_CONSTITUTION_V1.md` to every workout currently generated by `src/lib/seed-data.ts`.

Classification rules applied:

1. Core work alone does not create a Lift Day.
2. Mobility work alone does not create a Lift Day.
3. Mobility + Core is not automatically Hybrid.
4. Run Day remains Run Day even if mobility or core exists.
5. Lift Day remains Lift Day even if warmup and cooldown exist.
6. Long Run Day remains Long Run Day unless explicitly defined otherwise.
7. Hybrid Day requires two meaningful primary stimuli.
8. Sprint work counts as conditioning stress.
9. Long Run + Lift should be flagged.
10. Heavy Lift + Quality Run should be flagged.

## Recommended Fixes Before Planner Becomes Runtime Source of Truth

1. Add an explicit day-type classifier that resolves one primary day type before block construction.
2. Treat `core` as accessory/support work, not as a lift-defining category by itself.
3. Treat `mobility` as mobility/support work, never as a lift-defining category.
4. For `zone-2` workouts, classify run as primary and keep core/mobility supportive; do not create a workout/lift logging target from core support.
5. Add an explicit `conditioning` or `accessoryConditioning` block concept if sprint/finisher work should be displayed separately from both run and lift.
6. Keep Long Run Day protected: no lift block, no accidental hybrid, no extra conditioning beyond prescribed long run.
7. Make Hybrid Day opt-in through plan data or a clear plan rule; never infer Hybrid from support core/mobility.
8. Preserve run prescription mode exactly: duration stays duration, distance stays distance.
9. Add source-plan metadata for day type, primary stimulus, optional blocks, and explicit hybrid permission before importing future PDFs.
10. Promote the Phase 23C tests from audit-only failing validation to required green tests only after planner classification logic is updated.
